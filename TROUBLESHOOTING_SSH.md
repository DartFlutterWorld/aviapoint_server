# Устранение проблем с SSH подключением

## Проблема: Connection refused на порту 22

### Диагностика:

1. **Сервер отвечает на ping** ✅
   - IP адрес доступен: `83.166.246.205`
   - Сеть работает

2. **Порт 22 недоступен** ❌
   - SSH сервер не отвечает
   - Возможные причины:
     - SSH сервис остановлен
     - Порт изменен
     - Файрвол блокирует подключение
     - Сервер перезагружается

### Решения:

#### 1. Проверить через панель управления хостингом
- Войти в панель управления (Timeweb, если используется)
- Проверить статус сервера
- Проверить настройки файрвола
- Перезапустить SSH сервис через панель

#### 2. Проверить альтернативные порты
```bash
# Попробовать другие стандартные порты SSH
ssh -p 2222 root@83.166.246.205
ssh -p 22022 root@83.166.246.205
```

#### 3. Использовать веб-консоль хостинга
- Войти в панель управления
- Использовать веб-консоль (VNC/NoVNC)
- Проверить статус SSH:
  ```bash
  systemctl status ssh
  systemctl status sshd
  ```

#### 4. Перезапустить SSH через панель
- В панели управления найти опцию "Перезапустить SSH"
- Или через веб-консоль:
  ```bash
  sudo systemctl restart ssh
  sudo systemctl restart sshd
  ```

#### 5. Проверить файрвол
```bash
# Через веб-консоль
sudo ufw status
sudo iptables -L -n | grep 22
```

### Временное решение:

Если SSH недоступен, можно:
1. Использовать веб-консоль хостинга для управления
2. Использовать FTP/SFTP для передачи файлов (если доступен)
3. Обратиться в поддержку хостинга
